Topical Analysis on the SEC XBRL Filings

Posted on Tue 10 October 2017 in posts

Overview¶

This blog presents my attempt to discover topics and categorize the sections in the SEC XBRL filings using topic modeling technique. The discovered topics can help to facilitate querying filing sections by meaningful topics, for example searching all income tax related sections, or analyze filing sections based on their categories.

Collecting Data¶

The filings are collected from the SEC Edgar website. I have selected 10-K filings with period end date between 2016-04-01 and 2017-04-01. There are total of 5602 filings in the data set. Each filing is parsed with the python package Arelle to retrieve the title, and concepts from each section in the filing. A filing section, in XBRL terms, is a presentation group identified by an extended link role(ELR) URI. In this article, the word "ELR" is used interchangeably with the "section".

Cleaning and Preprocessing Text¶

Each section(ELR) in a filing has a title and a list of concepts. Each concept has a short text description called a label describling the meaning of the concept. We treat each section as a document composed of the text of the title and the concatenation of all concept labels. The title and concept labels are preprocessed with the following steps,

  1. Remove stop words. The stop words are common words that frequently appear in any document, and so serve no purpose of distinghishing documents. The common English stop words includs "the", "are", "about" etc. We have extended the English stop word set with the domain specific words such as "statement", "disclosure", "line", "item", "table", "axis" etc. These words appear in many section titles or concept labels and they bear insignificant meanings on the document.

  2. Remove non-word characters and parenthese/bracket enclosed content. The parenthese enclosed content, such as "(detail)", "(table)" etc, in section titles normally indicate the level of filing. The bracket enclosed content in concept labels indicate the type of concepts. They hold no semantic meanings on the section or concepts.

  3. Lemmatize words. Lemmatization normalizes the word representation. It turns a word to the base form of the word. For example, the plural word "revenues" is changed to the singular form "revenue". The lemmatization process reduces the word form variations and relates various word forms to the same base form.

Below is the preprocessing function,

# this pattern is used in processing elr description and concept labels to remove the brackets/parentheses enclosed content and # digits
remove_character_pattern = re.compile(r'\[.*\]|\(.*\)|\d')

# NLTK WordNetLemmatizer
lemma = WordNetLemmatizer()

def preprocess_text(text):
    # remove any content enclosed with brackets/parentheses such as [Table]|[Abstract], number, and parentheses
    text = re.sub(remove_character_pattern, ' ', text)

    # replace any non word character with space
    text = re.sub(r'\W', ' ', text)

    # tokenization and lemmatization
    words = [lemma.lemmatize(word) for word in text.lower().split() if word not in stop_word_set and len(word) >= 3]
    return words

Create Topic Model¶

What is topic modeling¶

Topic modeling is an unsupervised document clustering method. It is often used to discover topics in a document collection, and catagorize documents based on the discoved topics. The most polpular technique used for fitting a topic model is called LDA(Latent Dirichlet Allocation).

LDA assumes each document is generated by a topic distribution, and each topic is generated by a word distribution. That is, for each word in a document, you select a topic from a distribution over topics, and then pick up a word from the word distribution for that topic. For example, suppose the document collections has 4 topics "cash flows", "comprehensive income", "leases" and "debt". When generating a document, you first select a topic from a topic distribution for the document, say 60% for "cash flows", 20% for "comprehensive income", 10% for "leases" and "debt" respectively, if the selected topic is "cash flows", next you draw a word from the word distribution for the topic "cash flows". The document is created by repeating the process of each word generation. For a given number of topics and a collection of documents, the process of LDA is to estimate the topic distribution for each document, and the word distribution for each topic.

Creating a topic model¶

LDA is an unsupervised learning process, meaning that you don't need to provide a label for each document to train the model, but it requires users to provide the specific number of topics for the given collection of document. For this data collection, I have tried 30, 40, 50 and 60 topics, and choose 30 topics at last for the least topic overlaps and the highest level of interpretability.

The following code snippt creates the topic model using python gensim package,

num_topics = 30
class ELRCorpus(object):
    def __init__(self, elr_cursor, elr_dictionary):
        self.elr_cursor = elr_cursor
        self.elr_dictionary = elr_dictionary

    def __iter__(self):
        self.elr_cursor.rewind()
        for elr in self.elr_cursor:
            yield self.elr_dictionary.doc2bow(elr['ELRTitleWords'] + elr['ELRBaseLabelWords'] + elr['ELRExtLabelWords'])

def build_dict(elr_cursor):
    dictionary = corpora.Dictionary(elr['ELRTitleWords'] + elr['ELRBaseLabelWords'] + elr['ELRExtLabelWords'] for elr in elr_cursor)

    dictionary.filter_extremes(keep_n = 10000, no_below=10) # filter_extremes(no_below=5, no_above=0.5, keep_n=100000)
    dictionary.compactify()
    return dictionary

def create_model(elr_collection, num_topics, dictionary_path, corpus_path, tfidf_corpus_path, lda_model_path):
    elr_cursor = elr_collection.find()

    # build the dictionary for the words in ELR title and concept labels
    elr_dictionary = build_dict(elr_cursor)
    corpora.Dictionary.save(elr_dictionary, dictionary_path)

    # create a corpus for the ELRs by converting ELR documents into document term matrix
    elr_corpus = ELRCorpus(elr_cursor, elr_dictionary)
    # save the corpus in BleiCorpus format
    BleiCorpus.serialize(corpus_path, elr_corpus, id2word=elr_dictionary)

    # convert the document term frequency matrix into the inverse document term frequence matrix
    tfidf = gensim.models.tfidfmodel.TfidfModel(elr_corpus)
    tfidf_corpus = tfidf[elr_corpus]
    BleiCorpus.serialize(tfidf_corpus_path, tfidf_corpus, id2word=elr_dictionary)

    # Train LDA model from the inverse document term frequency matrix                                                  
    lda_model = gensim.models.LdaModel(tfidf_corpus, num_topics, id2word = elr_dictionary)
    lda_model.save(lda_model_path)
    return lda_model

Note: I have initially stored the data set in MongoDB. This code snippet reads data from MongoDB. It can be easily adapted to read from a CSV file. I have also saved the resulting dictionary, corpus, and LDA model to files for later use.

Assign topics to ELRs¶

Once we have the topic model, we can infer the topics for each filing section.

load dictionary, coprpus and LDA model¶

dictionary_path = '../models/elr_dictionary.dict'
tfidf_corpus_path = '../models/tfidf_corpus.lda-c'
lda_path = '../models/elr_lda_tfidf_' + str(num_topics) + '_topics.lda'

dictionary = gensim.corpora.Dictionary.load(dictionary_path)
tfidf_corpus = gensim.corpora.BleiCorpus(tfidf_corpus_path)
lda_model = gensim.models.ldamodel.LdaModel.load(lda_path)

Create document topic distribution dataframe from the LDA Model¶

gamma, _ = lda_model.inference(tfidf_corpus)
doc_topic_dists = gamma / gamma.sum(axis=1)[:, None]

Assign the most probable topic to each ELR¶

# elr_df is the pandas dataframe for the resulting dataset, for each section,we choose the topic with the highest probablity.
elr_df['TopicId'] = np.argmax(doc_topic_dists.values, axis=1)

Connecting LDA Topics with Accounting Standard Codification Topics¶

LDA model provides topic distributions for each document, but it does not provide labels for the topics. We can make some sense of these topics by the word distributions for topics.
Gets the top 10 words for each topic from the topic model topid word distributions

topic_word_dist = pd.DataFrame(columns = ['topic_id', 'word', 'prob'])
for topic_id in range(num_topics):
    for word, prob in lda_model.show_topic(topic_id, topn=10):
        topic_word_dist = topic_word_dist.append({'topic_id': topic_id, 'word': word, 'prob': prob}, ignore_index=True)

Based on the word distributions, and also the most common tags in each topic, I have attempted an interpretation by connecting these 30 LDA topics to the topics defined in the Accounting Standard Codification, and the sections defined in the US-GAAP taxonomy. Topical Analysis on Usages of Customized Tags¶

Once we have assigned a topic for each section, we can analyze the dataset based on the topics. One use case is to analyze the customized tags usage in terms of topics. The US filers are allowed to use company specific tags in their XBRL filings besides the standard tags defined in the US-GAAP. While these customized tags might be valuable for finding company specifics, they certainly increase difficulties for comparative analysis across company filings, and to some extent, defeat the very purpose of using standard vacabulary to enhance interprebablity. Therefore, it is helpful for standard setters and regularlators to estimate and understand the usages of these customized tags in the filings.

This code snippet calculates the number of sections(ELRs) that only use the standard tags for each topic.

topic_summary = pd.DataFrame(columns = ['topic_id', 'base_elrs', 'ext_elrs', 'total_elrs'])
topic_summary['topic_id'] = np.arange(30)
topic_summary['total_elrs'] = elr_df.groupby('TopicId').size().values
topic_summary['base_elrs'] = elr_df.loc[elr_df['ELRExtConcepts'].isnull()].groupby('TopicId').size().values
topic_summary['ext_elrs'] = topic_summary['total_elrs'] - topic_summary['base_elrs']

The visualization of the percentage of sections with only standard tags versus the sections with mixed standard and customized tags for each topic. hover over the circles and see the topics and percentage of sections that contain only standard tags. As can be seen from the visualization, most of topics are consisted of around 50% sections with only standard tags and 50% sections with mixed tags. But there are couple of topics with extreme cases.
